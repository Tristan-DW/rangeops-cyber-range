# Contributing to RangeOps

Thanks for helping improve RangeOps.

## Development setup

1. Install dependencies with `npm install`.
2. Start API with `npm run dev:api`.
3. Start web UI with `npm run dev -w @rangeops/web`.
4. Run checks with `npm run typecheck`.

## Pull request expectations

- Keep pull requests focused and small when possible.
- Include a short test plan in the PR description.
- Update docs when APIs or behavior change.
- Do not commit secrets or `.env` files.

## Commit style

Use clear, imperative commit messages:

- `feat: add run overview endpoint`
- `fix: validate lab creation payload`
- `docs: expand quick start guide`
